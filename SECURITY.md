# 🔒 Security Policy

Welcome to the **Neon Core AI** Security Policy.

We take security seriously and appreciate responsible disclosure of vulnerabilities.

---

## 🛡️ Supported Versions

The following versions currently receive security updates:

| Version            | Supported      |
| ------------------ | -------------- |
| Latest Release     | ✅ Yes          |
| Development Builds | ⚠️ Best Effort |
| Older Releases     | ❌ No           |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in Neon Core AI, please **do not disclose it publicly immediately**.

Instead:

1. 📧 Contact the development team privately.
2. 📝 Provide a detailed description of the issue.
3. 🔄 Include steps to reproduce the vulnerability.
4. 💻 Include screenshots, logs, or proof-of-concept code if applicable.

### Helpful Information

Please include:

* 🔍 Vulnerability type
* 📦 Affected version
* 🖥️ Operating system
* 📋 Steps to reproduce
* ⚠️ Potential impact

---

## ⏳ Response Timeline

We aim to:

* 📬 Acknowledge reports within 7 days
* 🔎 Investigate reported issues promptly
* 🔧 Release fixes as soon as practical
* 📢 Credit responsible reporters when appropriate

---

## 🚫 Please Do Not

* ❌ Access data that does not belong to you
* ❌ Attempt to disrupt services
* ❌ Perform denial-of-service attacks
* ❌ Publicly disclose vulnerabilities before a fix is available

---

## 🤝 Responsible Disclosure

We appreciate security researchers, developers, and community members who help improve Neon Core AI.

By reporting vulnerabilities responsibly, you help keep users safe and strengthen the project.

Thank you for helping make **Neon Core AI** safer for everyone.
